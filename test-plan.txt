ENG
 
1. Introduction:
The goal of testing is to gain an experience in software testing and finding defects intentionally implemented by developers.

2. Tests range:
System manual functional testing will be made.

There won't be efficiency and automatic testing (lack of resources).

3. Test object:
The test object will be the MrBuggy7 app downloadable from:

http://mrbuggy.pl/mrbuggy7/

4. Pass/fail criteria:
No criteria

5. Entry criteria:
Test software is downloaded and opens succesfully.

6. Exit criteria:
All test cases finished succesfully and confidense in enough experience gained from the process.

7. Test environment:
Desktop with Intel Core i7-8650U-CPU and 8 GB RAM

64-bit Windows 10 Pro.

8. Test Schedule:
- reviewing specifications

- preparing test cases

- executing test cases

- exploratory testing

- preparing test report

9. Test report:
- list of test cases statuses

- summary

- estimated quality of the product

- recommendations

10. Test tools:
- TestLink 1.9.20

- Postman

- Paint

- ShareX

- Jira

11. Defect management:
Reporting into Jira.

12. Roles and responsibilities:
Jakub Rzempała - preparing test plan, test cases, executing tests and preparing test report.

 

PL

Wykonane zostaną manualne testy fukcjonalne systemowe.

Nie zostaną wykonane testy wydajnościowe oraz testy automatyczne (brak zasobów).

3. Przedmiot testów:
Przedmiotem testów będzie aplikacja MrBuggy7 dostępna do pobrania ze strony:

http://mrbuggy.pl/mrbuggy7/

Brak.

6. Kryteria wyjściowe:
Wszystkie przypadki testowe wykonane pomyślnie.

7. Środowisko testowe:
Laptop Intel Core i7-8650U-CPU oraz 8 GB RAM

64-bit Windows 10 Pro.

8. Harmonogram testów:
- przegląd specyfikacji

- przygotowanie przypadków testowych

- sporządzenie raportu

9. Raport testowy:

- ocena jakości produktu

- rekomendacje

10. Narzędzia testowe:
- TestLink 1.9.20

- ShareX

- Jira

11. Zarządzanie błędami:
Zgłaszanie ich w projekcie na platformie Jira.
